# Neon Settings Menu Benchmark Brief

Starting from a clean copy of `examples/starter`, create a polished neon-style settings menu in
real C++ Dear ImGui. Use only the permitted Studio tools listed in `benchmark.json`; do not accept
human source edits or seed the attempt with completed reference-specific source.

The fixed 900 × 600 menu needs a branded header and icon action, grouped nested sidebar navigation,
two primary content panels, animated toggles, integer and float sliders, a keybind, combo, color
picker, and button. Use a custom font/icon set and a cohesive dark layered panel system with a neon
accent. Opening, hover, selection, toggle, slider, and dropdown motion must be visible and stable.

Iterate through real build, preview, inspection, interaction, comparison, deterministic capture,
and verified native export operations. Every required interaction needs a stable Studio widget ID.
Required controls must be custom-drawn or intentionally composed from custom-drawn foundations.
No unresolved clipping, overlap, duplicate-ID, invalid geometry, or invalid draw diagnostics may
remain.

The reference image is supplied separately as an evaluation input because its redistribution rights
must be established by the evaluator. Its presentation bloom and perspective are inspiration, not
portable rendering requirements. Visual similarity is diagnostic and never overrides interaction,
determinism, inspection, or parity requirements.
