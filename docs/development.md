# Development Workflow

## Quality gate

Run the complete supported native gate from a Visual Studio developer PowerShell:

```powershell
npm run validate
```

Run narrower checks during development:

```powershell
npm run schemas:check
npm run licenses:check
npm run typecheck
npm run lint
npm run format:check
npm run format:cpp:check
npm run test:ts
npm run test:cpp
npm run test:comparison
npm run test:phase2
npm run test:studio
```

## Generated contracts

Use `npm ci` for a clean dependency installation. Edit canonical JSON Schemas, then regenerate:

```powershell
npm run schemas:generate
npm run test:ts
```

Never edit `schemas/generated/*.generated.ts` manually. CI checks the generated bytes against the
canonical schemas.

## C++ configuration

Windows native commands used by the wrapper are:

```powershell
cmake --preset native-msvc
cmake --build --preset native-msvc
ctest --preset native-msvc
```

Studio-owned C++ compiles as C++20 with warnings as errors. Build output is isolated under
`build/`.

## Phase 1 preview and parity workflow

Build and test the real WebAssembly/WebGL2 preview from an Emscripten-enabled PowerShell:

```powershell
. .\.tools\emsdk\emsdk_env.ps1
.\toolchain\emscripten\build-preview.ps1
npm run test:browser
```

For interactive development, run `npm run preview:serve` and open
`http://127.0.0.1:4173`. The Studio shell and preview use separate loopback origins.

Build the Win32/DirectX 11 host, capture the same starter source, and compare:

```powershell
.\toolchain\run-native.ps1
.\toolchain\capture-native.ps1
npm run compare:captures
```

`run-native.ps1` launches the persistent, input-enabled `--interactive` mode. The capture script
omits that flag, uses fixed frame timing, writes canonical artifacts, and exits automatically.

Generated captures, metadata, reports, and PNG differences remain under ignored `out/`
directories. Do not update a visual expectation without reviewing its diff and provenance.

## Phase 2 edit/build workflow

```powershell
. .\.tools\emsdk\emsdk_env.ps1
npm run studio
npm run agent-adapter
```

The service snapshots canonical inputs under `.studio/builds/<buildId>/snapshot`, copies the selected
snapshot into a stable CMake input directory, recompiles the changed starter translation unit, and
links a new immutable preview. Compile, link, artifact promotion, and smoke durations are recorded
separately. The stable Dear ImGui/backend object cache is content-verified before reuse.

Run the service and browser gates after changing filesystem, revision, build, preview, authentication,
or editor code:

```powershell
npm run test:phase2
npm run test:studio
```

## Troubleshooting

- **Version verification fails:** install the exact version in `toolchain/toolchain.json` or use
  the documented bootstrap.
- **`cl.exe` is missing:** open a Visual Studio 2022 developer PowerShell for x64.
- **Generated types are stale:** run `npm run schemas:generate`.
- **License inventory is stale:** run `npm run licenses:generate` after intentionally updating dependencies with `npm install`.
- **CMake cache is invalid:** run `npm run clean`, then configure again.
- **`emcc` is missing:** bootstrap Emscripten and dot-source `.tools/emsdk/emsdk_env.ps1`.
